# Positioning Canvas

Use this to structure a positioning exercise. Fill every box before
writing prose. Prose hides weak thinking; the canvas exposes it.

---

## 1. For (target user)

**Specific persona**, not a category. "Korean B2B SaaS PMs at 2-5 yrs"
beats "PMs".

- Primary: {who}
- Secondary: {who}
- NOT for: {who — explicitly exclude}

## 2. Who need (the Job to be done)

The Job, written as: "When {situation}, I want to {motivation}, so I can
{outcome}".

- Primary Job: {one line}
- Adjacent Jobs we also support: {list 1-2 max}

## 3. Our product is a (category)

Categories the user ALREADY understands. Inventing a new category is
expensive ("the first fill-in-your-buzzword platform"). Be honest about
the existing category first.

- Category: {e.g., "AI coding agent framework"}
- Sub-category (if any): {e.g., "for PM workflows"}

## 4. That (primary benefit / proof point)

The one thing you deliver better than alternatives. Specific and provable.

- Benefit: {one line, verb-first}
- Proof: {metric, user quote, demo — something concrete}

## 5. Unlike (primary alternative — named)

The ONE competitor or incumbent to contrast against. "Other tools" is not
a position; name names.

- Alternative: {named competitor / incumbent / default behavior}
- How they solve this today: {one line}
- Why they can't solve it as well as we do: {structural reason, not opinion}

## 6. Our unique capability (proof of #4 vs #5)

What we can do that the alternative structurally cannot? Temporary product
advantages are not capabilities.

- Capability: {specific}
- Why structural: {moat reason — data, network, distribution, insight, cost}

## Filled example (SuperPM itself)

1. **For**: PMs using Claude Code / Cursor who want a virtual PM team
2. **Who need**: "When drafting PM artifacts or running PM rituals, I want
   to invoke the right PM perspective on demand, so I can work at senior-PM
   quality without a team."
3. **Our product is**: a Role-based Skills collection for AI coding agents
4. **That**: mirrors the eight PM roles a single PM has to fill, with
   composable Tools / Modes / Adversarial / Meta layers
5. **Unlike**: flat PM prompt libraries (claude-skills, Product-Manager-Skills)
   that give you hundreds of prompts to choose from
6. **Our unique capability**: a multi-layer architecture where Roles /
   Scenarios / Tools / Modes compose, so users invoke "who / when / how"
   independently. Structurally harder for a flat prompt list to replicate
   without rebuilding the taxonomy.

---

## Anti-patterns

- "For everyone" → not positioning, that's avoidance
- Category you invented → raises the adoption bar
- Vague "better than competitors" → name them and structure why
- Benefits that any competitor could also claim → not differentiation
- Capability that is a feature, not a moat → temporary advantage
